# 《你不知道的 JavaScript》

# 作用域

存储变量，并且之后对这个值可以进行访问或修改，这套规则被称为作用域。

JavaScript 事实上是一门编译语言，只是与传统的编译语言不同，他不是提前编译的，编译结果也不在分布式系统中进行移植。

# 编译原理

传统的编译语言流程，程序中的一段源代码在执行之前会经历三个步骤，统称为：编译。

  • 分词/词法分析（Tokenizing/Lexing）

    这个过程会将字符组成的字符串分解成有意义的代码，这些代码块被称为词法单元。

    分词和词法分析之间主要差异在于词法单元的识别是通过有状态还是无状态的方式进行的。比如：如果词法单元生成器在判断 a 是一个独立的词法单元还是其他词法单元的一部分时，调用的是有状态的解析规则，那么这个过程就被称为词法分析。如果这个词法单元没有状态，那么这个过程就被称为分词。

  • 解析/语法分析（Parsing）

    将词法单元流（数组）转换成一个由元素逐级嵌套所组成的代表了程序语法结构的树，这个树被称为抽象语法树（Abstract Syntax Tree，AST）

  • 代码生成

    将 AST 转换成可执行代码的过程被称为代码生成。简单来说：就是讲代码的 AST 树转换为一组机器指令。

JavaScript 引擎的编译原理要比这个复杂的多，对 JavaScript 来说，编译过程发生在代码执行前的几微秒的时间内。

# 理解作用域

  • 引擎：从头到尾负责整个 JavaScript 程序的编译及执行过程
  • 编译器：引擎的好朋友之一，负责语法分析及代码生成等
  • 作用域：引擎的好朋友之一，负责收集并维护由所有声明的标识符（变量）组成的一系列查询，并实施一套非常严格的规则，确定当前执行的代码对这些标识符的访问权限。

对于var a = 2，编译器会做如下处理：

  1. 遇到var a，编译器会询问作用域是否有一个该名称的变量存在与同一个作用域的集合中。如果是，则编译器会忽略该声明，继续编译；否则会在当前作用域的集合中声明一个 a 变量。
  2. 接下来编译器会为引擎生成运行时所需的代码，这些代码被用来处理a = 2这个赋值操作，引擎运行时会先询问作用域，在当前的作用域集合中是否存在 a 变量，如果是，引擎就是使用这个变量，如果否，引擎继续查找该变量。如果引擎最终找到了 a，就会将 2 赋值给它，否则，抛出异常。

引擎为变量进行查找的方式分为 LHS 查询、RHS 查询。

当变量出现在赋值操作的左侧时进行 LHS 查询，出现在右侧时进行 RHS 查询。

RHS 查询是查找某个变量的值，LHS 查询是试图查找到变量的容器本身，从而可以对其赋值。

在变量还没有声明，这两种查询的行为是不一样的。

当引擎执行 RHS 查询，如果在所有作用域都找不到变量，则会抛出 ReferenceError 异常。相比之下，当引擎执行 LHS 查询时，如果在所有作用域都找不到变量，全局作用域就会创建一个具有该名称的变量，并将其返回给引擎，前提是程序运行在非严格模式下。

ES5 中创建了严格模式，在严格模式下禁止自动或隐式地创建全局变量。

ReferenceError 同作用域判别失败相关，而 TypeError 则代表作用域判别成功了，但是对结果的操作是非法或不合理的。

# 词法作用域

作用域有两种工作模型，第一种是最普遍的词法作用域，另一种是动态作用域。

# 词法阶段

词法作用域意味着作用域是由书写代码时函数声明的位置来决定的。

在多层的嵌套作用域中可以定义同名的标识符，这叫做遮蔽效应（内部的标识符遮蔽了外部的标识符）。

全局变量会自动称为全局对象（比如浏览器中的 window 对象）的属性，因此可以不直接通过全局对象的词法名称，而是间接地通过对全局对象属性的引用来对其进行访问。

window.a;

**通过这种技术可以访问那些被同名变量所遮蔽的全局变量，**但非全局变量如果被遮蔽了，无论如何都无法被访问到。

词法作用域只与函数被声明时所处的位置有关。

# 欺骗语法

词法作用域在函数声明的位置来定义，修改词法作用域的行为叫做欺骗语法。

欺骗语法作用域会导致性能下降。

有两种方法可以实现

# eval

eval() 函数可以接受一个字符串为参数，并将其中的内容视为好像在书写时就存在于程序中这个位置的代码。

function foo(str, a) {
  eval(str); // 欺骗
  console.log(a, b);
}

var b = 2;
foo("var b = 3", 1); // 1, 3

上例对foo() 的词法环境进行了修改。实际上这段代码在 foo 内部创建了一个变量 b，并遮蔽了外部作用域的同名变量。

在严格模式的程序中，eval() 在运行时有自己的词法作用域，意味着其中的声明无法修改所在的作用域。

function foo(str) {
  "use strict";
  eval(str);
  console.log(a); // ReferenceError: a is not defined
}

foo("var a = 2");

JS 中还有一些其他功能效果和eval(...) 相似，setTimeout(...) 和setInterval(...) 第一个参数可以是字符串，字符串的内容可以被解释为一段动态生成的函数代码。不过这些功能已过时，不建议使用。

new Function(...) 函数的行为也很类似，最后一个参数可以是接受代码的函数，并将其转换为动态生成的函数（前面的参数是这个新生成的函数的形参）。

let sum = new Function("a", "b", "return a + b");

alert(sum(1, 2)); // 3
# with

with 是重复引用同一个对象中的多个属性的快捷方式，可以不需要重复引用对象本身。

var obj = {
  a: 1,
  b: 2,
  c: 3,
};

// 单调乏味的重复
obj.a = 2;
obj.b = 3;
obj.c = 4;

// 简单的快捷方式
with (obj) {
  a = 3;
  b = 4;
  c = 5;
}

with 实际上是将一个没有或有多个属性的对象处理为一个完全隔离的词法作用域，这个对象的属性会被处理为定义在这个词法作用域的词法标识符。

function foo(obj) {
  with (obj) {
    a = 2;
  }
}

var o1 = {
  a: 3,
};
var o2 = {
  b: 3,
};

foo(o1);
console.log(o1.a); // 2

foo(o2);
console.log(o2.a); // undefined
console.log(a); // 2 a被泄露到全局作用域了

eval 函数接受含有一个或多个声明的代码，修改所在的作用域，而 with 声明实际上根据传递给它的对象凭空创建一个全新的词法作用域。

当我们传递 o1 给 with 时，with 声明一个新的作用域 o1，这个作用域中含有 o1.a 属性，但当我们查看 o2 作用域时，没有 a 标识符，所以进行正常的 LHS 查找，在上一个作用域中查找，都没有找到标识符 a，最后在全局作用域自动创建一个全局变量。

注意：尽管 with 可以将一个对象处理为词法作用域，但是这个块内部正常的 var 声明不会被限制在这个块作用域中，而是被添加到 with 所处的函数作用域中。

# 函数作用域与块作用域

属于这个函数的所有变量，都可以在整个函数范围内使用及复用。

# 函数作用域

在任意代码片段外部添加包装函数，就可以将内部的变量和函数定义隐藏起来，外部作用域无法访问包装函数内部的任何内容，但是会增加函数名以及调用函数。JS 给我们提供了更好的方案

var a = 2;

(function foo() {
  var a = 3;
  console.log(a); // 3
})();
console.log(a); // 2

  • (function foo(){}) 会被当做一个函数表达式来执行

  • function foo(){} 会被当做一个函数声明来执行

函数表达式和函数声明的区分：

如果 function 是声明中的第一个词，就是函数声明，否则就是函数表达式。

函数表达式和函数声明最大的区别是他们的名称标识符将会绑定在何处，(function foo(){...}) 的名称标识符会被绑定在函数表达式自身的函数中，只能在...中被访问，而function foo(){}的名称标识符会被绑定在所在作用域中。

# 匿名和具名

因为function(){..} 没有名称标识符，这叫做函数表达式，函数表达式可以是匿名的，而函数声明则不可以省略函数名。

# 立即执行函数表达式
var a = 2;

(function foo() {
  var a = 3;
  console.log(a); // 3
})();
console.log(a); // 2

由于函数被包含在一对括号内部，因此成为了一个表达式，通过在末尾加上另一个括号可以立即执行这个函数。IIFE：立即执行函数表达式

另一个常用表达式是(function(){...}())。

IIFE 的另一个非常普遍的用法是把它们当做函数调用并传递参数进去。

var a = 2;

(function IIFE(global) {
  var a = 3;
  console.log(a); // 3
  console.log(global.a); // 2
})(window);

console.log(a); // 2

IIFE 还有一种变化的用途是倒置代码的运行顺序，将需要传递运行的函数放在第二位，在 IIFE 执行之后当做参数传递进去。

var a = 2;

(function IIFE(def) {
  def(window);
})(function def(global) {
  var a = 3;
  console.log(a); // 3
  console.log(global.a); // 2
});

# 块作用域

  • 用 with 对象创建出的作用域就是块级作用域。

  • try...catch...的 catch 分句创建的作用域也是块级作用域，其中声明的变量只在 catch 内部有效。所以在同一个作用域 try 出多个错误，catch 的参数命名可以一直为 err。

  • Let

    • 提升是指声明会视为存在于其所出现的作用域的整个范围内
    • 但是 let 进行的声明不会在块作用域中进行提升，声明的代码被运行之前，声明不存在

  • const

# 提升

编译的一部分工作就是找到所有的声明，并用合适的作用域将它们关联起来。

var a = 2，JS 会将其看成两个声明，var a and a = 2，第一个声明是在编译阶段进行的，第二个赋值声明会被留在原地等待执行阶段。

变量和函数声明从代码中被提升到了最上面，叫做提升。只要声明本身会被提升。函数声明提升是提升全部。

函数声明会被提升，函数表达式的赋值操作并不会被提升。

# 函数优先

函数声明和变量声明都会被提升，函数会首先被提升，然后才是变量。

foo(); // 1

var foo;

function foo(){
  console.log(1)；
}

foo = function(){
  console.log(2);
}

尽管 var foo 出现在 function foo 的声明之前，但他是重复声明，所以被忽略了，因为函数声明会被提升到普通变量之前。

尽管重复的 var 声明会被忽略，但是出现在后面的函数声明还是可以覆盖前面的。

foo(); // 3

function foo(){
  console.log(1)；
}

foo = function(){
  console.log(2);
}

function foo(){
  console.log(3);
}

普通块内部的函数声明通常会被提升到所在作用域的顶部，这个过程不会被条件判断所控制

foo(); // 'b'

var a = true;
if (a) {
  function foo() {
    console.log("a");
  }
} else {
  function foo() {
    console.log("b");
  }
}

# 作用域和闭包

接下来的内容是对作用域工作原理相关基础知识深入的理解。

闭包并不是一个需要学习的语法或模式，闭包是基于词法作用域书写代码时所产生的自然结果。甚至不需要有意识的创建闭包。

当函数可以记住并访问所在的词法作用域时，就产生了闭包，即使函数是在当前词法作用域之外执行。

词法作用域的查找规则，是闭包的一部分。

function foo() {
  var a = 2;

  function bar() {
    console.log(a);
  }

  return bar;
}

var baz = foo();
baz(); // 2 闭包的效果

在 foo 执行之后，通常会期待 foo 的整个内部作用域都被销毁，因为引擎有垃圾回收机制用来释放不再使用的内存空间。而闭包的神奇之处可以阻止这件事情的发生，事实上，内部作用域依然存在，因为没有被回收。

bar 持有对 foo 内部作用域的引用，而这个引用就叫做闭包。

无论使用何种方式对函数类型的值进行传递，当函数在别处被调用时都可以观察到闭包。

function foo() {
  var a = 2;

  function baz() {
    console.log(a); // 2
  }

  bar(bza);
}

function bar(fn) {
  fn(); // 这就是闭包
}

传递函数也可以是间接的

var fn;

function foo() {
  var a = 2;

  function baz() {
    console.log(a);
  }

  fn = baz;
}

function bar() {
  fn(); // 产生闭包
}

foo();

bar();

无论通过何种手段将内部函数传递到所在的词法作用域以外，都会持有对原始定义作用域的引用，无论和何处执行这个函数都会使用闭包。

function wait(message) {
  setTimerout(function timer() {
    console.log(message);
  }, 1000);
}

wait("hello, closure");

将一个内部函数传递给了定时器，wait 执行 1000 毫秒后，内部作用域并不会消失，timer 具有涵盖 wait 作用域的闭包，保有对变量的引用。

使用了回调函数就使用了闭包。

回调函数，被作为实参传递给函数，且在外部函数中被调用的函数

# 循环和闭包

for (var i = 1; i <= 5; i++) {
  setTimeout(function timer() {
    console.log(i);
  }, i * 1000);
}

延迟函数的回调在循环结束时才执行。所以会每隔一秒依次输出 5 个 6。

我们试图假设循环中的每个迭代在运行时都会给自己捕获一个 i 的副本。但是根据作用域的原理，实际情况是尽管循环中的五个函数是在各自的迭代中分别定义的，但是它们都被封闭在了一个共享的全局作用域中，因此实际上只有一个 i。

IIFE 会通过声明并立即执行一个函数来创建作用域。

for (var i = 1; i <= 5; i++) {
  (function () {
    setTimeout(function timer() {
      console.log(i);
    }, i * 1000);
  })();
}
// 依然不行，虽然有自己的词法作用域，但是没有变量i，还是会按照规则向上依次查找i

for (var i = 1; i <= 5; i++) {
  (function (j) {
    setTimeout(function timer() {
      console.log(j);
    }, j * 1000);
  })(i);
}
// 正常工作，因为将j传入了IIFE，所以相当于在词法作用域内部创建了一个j变量存储i

# 模块

模块利用了闭包。

function CoolModule() {
  var something = "cool";
  var another = [1, 2, 3];

  function doSomething() {
    console.log(something);
  }
  function doAnother() {
    console.log(another.join(" ! "));
  }

  return {
    doSomething: doSomething,
    doAnother: doAnother,
  };
}

var foo = CoolModule();

foo.doSomething(); // cool
foo.doAnother(); // 1 ! 2 ! 3

这个模式在 JS 中被称为模块。最常见的实现模块模式的方法通常被称为模块暴露。

将这个对象类型的返回值看作本质上是模块的公共 API。

从模块中返回一个实际的对象并不是必须的，也可以直接返回一个内部函数，例如 jQuery，jQuery 本身就是函数。（函数本身就是对象，所以可以拥有属性）

模块模式需要具备两个必要条件。

  1. 有外部的封闭函数，外部的函数必须至少被调用一次（每次调用都会创建一个新的模块实例）
  2. 封闭函数必须返回至少一个内部函数，这样内部函数才能在私有作用域中形成闭包，并且可以访问或者修改私有的状态。
# 未来的模块机制

在模块系统进行加载时，ES6 会将文件当做独立的模块来处理。每个模块都可以导入其他模块或特点的 API 成员，也可以导出自己的 API 成员。

  • 基于函数的模块不是一个能被静态识别的模式，编译器无法识别，API 只在运行的时候才会被考虑进来。所以可以在运行时修改一个模块的 API。
  • ES6 模块 API 是静态的，API 不会在运行时改变。编译器知道这一点，因此可以在编译期检查对导入模块的 API 成员的引用是否真实存在。如果 API 引用不存在，编译器会抛出错误。

ES6 模块必须被定义在独立的文件中，浏览器或引擎有一个默认的模块加载器，可以在导入模块时同步加载模块文件。

// bar.js
function hello(who){
  return 'Let me introduce: ' + who;
}

export hello;

// foo.js
import hello from 'bar';

var hungry = 'hippo';

function awesome(){
  console.log(
    hello(hungry).toUpperCase()
  )
}

export awesome;

// baz.js
module foo from 'foo';
module bar from 'bar';

console.log(bar.hello('rhino')) // Let me introduce: rhino

foo.awesome(); // LET ME INTRODUCE: HIPPO

import 可以将一个模块中的一个或多个 API 导入到当前作用域中，并分别绑定在一个变量上。

module 会将整个模块的 API 导入并绑定到一个变量上。

export 会将当前模块的一个标识符导出为公共 API。
