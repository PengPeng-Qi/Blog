# 类继承
## extends
`extends` 将`Rabbit.prototype.[[Prototype]]` 设置为`Animal.prototype`，所以，在`Rabbit.prototype` 中找不到的方法，就会从`Animal.prototype` 中获取该方法。
```js {1,12}
class Animal {
  constructor(name) {
    this.speed = 0;
    this.name = name;
  }
  run(speed) {
    this.speed = speed;
    alert(`${this.name} runs with speed ${this.speed}.`);
  }
}

class Rabbit extends Animal {
  hide() {
    alert(`${this.name} hides!`);
  }
}

let rabbit = new Rabbit("White Rabbit");

rabbit.run(5); // White Rabbit runs with speed 5.
rabbit.hide(); // White Rabbit hides!
```
JS内建对象也同样使用了原型继承，例如`Date.prototype.[[Prototype]]` 是`Object.prototype`  
  
`extends` 后允许任意表达式：
```js {7}
function f(phrase) {
  return class {
    sayHi() { alert(phrase); }
  };
}

class User extends f("Hello") {
  // ...   
}

new User().sayHi(); // Hello
```
这里`class User` 继承自`f("Hello")` 的结果。
## 重写方法
- `super.method(...)` 调用一个父类方法
- `super(...)` 调用一个父类`constructor`(只能在我们的`constructor`中) 
  
`rabbit` 在停下来的时候自动`hide`：
```js {25,28}
class Animal {

  constructor(name) {
    this.speed = 0;
    this.name = name;
  }

  run(speed) {
    this.speed = speed;
    alert(`${this.name} runs with speed ${this.speed}.`);
  }

  stop() {
    this.speed = 0;
    alert(`${this.name} stands still.`);
  }

}

class Rabbit extends Animal {
  hide() {
    alert(`${this.name} hides!`);
  }

  stop() {
    super.stop(); // 调用父类的 stop
    this.hide(); // 然后 hide
  }
}

let rabbit = new Rabbit("White Rabbit");

rabbit.run(5); // White Rabbit 以速度 5 奔跑
rabbit.stop(); // White Rabbit 停止了。White rabbit hide 了！
```
箭头函数没有`super`，如果被访问，他就会从外部函数获取：
```js
class Rabbit extends Animal {
  stop() {
    setTimeout(() => super.stop(), 1000); // 1 秒后调用父类的 stop
  }
}
```
箭头函数中的`super` 与`stop()` 中的是一样的，所以它能按预期工作。  
  
如果我们在这里指定一个普通函数，就会抛出错误：
```js
// 意料之外的 super
setTimeout(function() { super.stop() }, 1000);  // error
```
## 重写constructor
如果**一个类拓展另一个类**并没有`constructor`，那么将生成下面这样的**空`constrcutor`**：
```js {3,5}
class Rabbit extends Animal {
  // 为没有自己的 constructor 的扩展类生成的
  constructor(...args) {
    super(...args);
  }
}
```
我们给`Rabbit` 添加一个自定义的`constructor`，下例会发生错误：
```js
class Animal {
  constructor(name) {
    this.speed = 0;
    this.name = name;
  }
  // ...
}

class Rabbit extends Animal {
  constructor(name, earLength) {
    this.speed = 0;
    this.name = name;
    this.earLength = earLength;
  }

  // ...
}

let rabbit = new Rabbit("White Rabbit", 10); // Error: this is not defined.
```
**因为继承类的`constructor` 必须调用`super(...)`，且一定要在使用`this` 之前调用。**  
  
继承类(派生构造器)的构造函数与其他函数之前是有区别的。派生构造器具有特殊的内部属性 `[[ConstructorKind]]:"derived"`  
  
该标签会影响`new` 行为：
- 当通过`new` 执行一个常规函数时，他会创建一个空对象，并将这个空对象赋值给`this`。
- 当继承`constructor` 执行时，不会执行此操作，它期望父类的`constructor` 来完成这项工作。所以，必须调用`super` 才能执行父类的`constructor`。

## 重写类字段
**父类构造器总是使用父类的字段**
```js {14}
class Animal {
  name = 'animal';

  constructor() {
    alert(this.name);
  }
}

class Rabbit extends Animal {
  name = 'rabbit';
}

new Animal(); // animal
new Rabbit(); // animal
```
```js {18}
class Animal {
  showName() {  // 而不是 this.name = 'animal'
    alert('animal');
  }

  constructor() {
    this.showName(); // 而不是 alert(this.name);
  }
}

class Rabbit extends Animal {
  showName() {
    alert('rabbit');
  }
}

new Animal(); // animal
new Rabbit(); // rabbit
```
**类字段是这样初始化的**：
- 对于基类（还未继承任何东西的那种），在构造函数调用前初始化。
- 对于派生类，在`super()` 后立刻初始化。

在我们的例子中，`Rabbit` 是派生类，里面没有`constructor()`。正如先前所说，这相当于一个里面只有`super(...args)` 的空构造器。  
  
所以，`new Rabbit()` 调用了`super()`，因此它执行了父类构造器，并且（根据派生类规则）只有在此之后，它的类字段才被初始化。**在父类构造器被执行的时候，`Rabbit` 还没有自己的类字段**，这就是为什么`Animal` 类字段被使用了。
