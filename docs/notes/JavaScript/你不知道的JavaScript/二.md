## 《你不知道的 JavaScript》

## This 和对象原型

### 关于 This

This 关键字被自动定义在所有函数的作用域中。

This 是在函数被调用时发生的绑定，并不是在编写时绑定，他的上下文取决于函数调用时的各种条件。This 的绑定（动态作用域）和函数声明的位置（词法作用域）没有任何关系，只取决于函数的调用方式。

当一个函数被调用时，会创建一个活动记录（执行上下文），这个记录会包含函数在哪里被调用（调用栈）、函数的调用方式、传入的参数等信息。This 就是这个记录的一个属性，会在函数执行的过程中用到。

### This 全面解析

#### 绑定规则

##### 默认绑定

```js
function foo() {
  console.log(this.a);
}

var a = 2;
foo(); // 2
```

调用`foo()`,`this.a` 被解析成了全局变量，函数调用时应用了 this 的默认绑定，this 指向全局对象。`foo()` 直接使用不带任何修饰的函数引用进行调用，就会使用默认绑定。

如果使用严格模式，则不能将全局对象用于默认绑定，因此 this 会绑定到 undefined

```js
function foo() {
  "use strict";
  console.log(this.a);
}

var a = 2;
foo(); // TypeError: this is a undefined
```

在严格模式下**调用`foo()` 不影响默认绑定**：

```js
function foo() {
  console.log(this.a);
}

var a = 2;

(function () {
  "use strict";

  foo(); // 2
})();
```

##### 隐式绑定

另一个考虑的规则是调用位置是否有上下文对象，或者说是否被某个对象拥有或包含。

```js
function foo() {
  console.log(this.a);
}

var obj = {
  a: 2,
  foo: foo,
};

obj.foo(); // 2
```

隐式绑定规则会把函数调用中的 this 绑定到这个上下文对象。因为调用`foo()` 时 this 被绑定到 obj。

对象属性引用链中只有上一层或最后一层在调用位置中起作用。

```js
function foo() {
  console.log(this.a);
}

var obj2 = {
  a: 42,
  foo: foo,
};

var obj1 = {
  a: 2,
  obj2: obj2,
};

obj1.obj2.foo(); // 42
```

一个常见的 this 绑定问题是被隐式绑定的函数会丢失绑定对象，会应用默认绑定。从而把 this 绑定到全局对象或 undefined 上，取决于是否是严格模式。

```js
function foo() {
  console.log(this.a);
}

var obj = {
  a: 2,
  foo: foo,
};

var bar = obj.foo; // 函数别名
var a = "oops, global"; // a是全局对象的属性

bar(); // 'oops, global'
```

bar 是`obj.foo` 的一个引用，但是实际上，引用的是 foo 函数本身，因此`bar()` 是一个不带任何修饰的函数调用，使用默认绑定。

参数传递是一种隐式赋值。

JS 中`setTimeout()` 函数实现和下面的伪代码类似：

```js
function setTimeout(fn, delay) {
  // 等待delay 毫秒
  fn(); // <-- 调用位置
}
```

##### 显示绑定

`call() 、apply()` 这两个函数接受第一个参数为对象，是给 this 准备的。我们称之为显示绑定。

> 如果传入一个原始值来当做 this 的绑定对象，这个原始值会被转换为它们的对象形式。（`new String(...) new Boolean(..)...等`）

```js
function foo() {
  console.log(this.a);
}

var obj = {
  a: 2,
};

var bar = function () {
  foo.call(obj);
};

bar(); // 2
setTimeout(bar, 100); // 2

// 硬绑定的bar不可能再修改它的this
bar.call(window); // 2
```

##### new 绑定

构造函数是一些使用 new 操作符时被调用的函数。不属于某个类，也会实例化一个类。

new 调用函数，会执行下面的操作：

1. 创建一个全新的对象
2. 将这个对象执行`[[Prototype]]` 连接
3. 将这个新对象的 this 绑定到函数调用的 this
4. 如果这个函数没有返回其他对象，那么 new 表达式中的函数调用会自动返回这个新对象。

#### 优先级

new 绑定>显示绑定（包含硬绑定）>隐式绑定>默认绑定

在 new 中使用**硬绑定函数**，目的是预先设置函数的一些参数

```js
function foo(p1, p2) {
  this.val = p1 + p2;
}

var bar = foo.bind(null, "p1");
var baz = new bar("p2");

baz.val; // p1p2
```

#### 绑定例外

##### 被忽略的 this

**如果`null`或者`undefined`作为 this 的绑定对象传入`call、apply`或者`bind`，这些值在调用时会被忽略，实际应用的是默认绑定规则。**

```js
function foo(a, b) {
  console.log("a: " + a + "b: " + b);
}

foo.apply(null, [2, 3]); // a: 2 b: 3

var bar = foo.bind(null, 2);
bar(3); // a: 2 b: 3
```

一种更安全的方式是传入一个特殊的对象，把 this 绑定到这个特殊的对象上。

```js
function foo(a, b) {
  console.log("a: " + a + "b: " + b);
}

var ø = Object.create(null);

foo.apply(ø, [2, 3]); // a: 2 b: 3

var bar = foo.bind(ø, 2);
bar(3); // a: 2 b: 3
```

##### 间接引用

创建一个函数的间接引用，在这种情况下，调用这个函数会应用默认绑定规则。

```js
function foo() {
  console.log(this.a);
}

var a = 2;
var o = { a: 3, foo: foo };
var p = { a: 4 };

o.foo(); // 3
(p.foo = o.foo)(); // 2
```

**`p.foo = o.foo` 的返回值是目标函数的引用，因此调用位置是`foo()` 而不是`p.foo() 或 o.foo()`，所以，会应用默认绑定。**

##### 软绑定

硬绑定把 this 强制绑定到指定的对象，但是有一个问题，硬绑定大大的降低函数的灵活性，使用硬绑定之后就无法使用隐式绑定或者显式绑定来修改 this。

如果给默认绑定指定一个全局对象和 undefined 以外的值，那么就可以实现和硬绑定相同的效果，同时保留隐式绑定或者显式绑定修改 this 的功能。

#### 箭头函数

箭头函数不使用 this 的四种标准规则，而是根据外层作用域来决定 this。

```js
function foo() {
  return (a) => {
    console.log(this.a);
  };
}

var obj1 = {
  a: 2,
};

var obj2 = {
  a: 3,
};

var bar = foo.call(obj1);
// 给foo指定this为obj1，返回一个箭头函数

bar.call(obj2);
// 2 给箭头函数指定this为obj2，但是箭头函数的this为上一个作用域的this，所以箭头函数的this是foo的this：obj1
```

箭头函数最常用于回调函数中，例如事件处理器或者定时器：

```js
function foo() {
  setTimeout(() => {
    console.log(this.a);
  }, 100);
}

var obj = {
  a: 2,
};

foo.call(obj); // 2
```

### 对象

JavaScript 有一些对象子类型，通常被称为内置对象。

如果在`string`上执行一些属性操作，**引擎会自动把字面量转换成`String` 对象**，也就是说不需要显式创建一个对象，所以可以访问属性和方法。
