# 执行上下文
当执行一个函数的时候，就会创建一个执行上下文，并且压入上下文栈。当函数执行完毕的时候，就会将函数的执行上下文从栈中弹出。  
  
对于每个执行上下文，都有三个重要属性：
- 变量对象
- 作用域链
- `this`

## 变量对象
**变量对象**是与执行上下文相关的数据作用域，**存储了在上下文中定义的变量和函数声明**。  
  
因为不同执行上下文的变量对象不同，所以我们来聊一下**全局上下文的变量对象**和**函数上下文的变量对象**。  
### 全局上下文
**全局对象**：全局上下文的对象就是全局对象
1. 可以通过`this` 引用，在客户端JavaScript 中，全局对象就是`Window` 对象。
```js
console.log(this);  // Window
```
2. 全局对象是由`Object` 构造函数构造出来的一个实例对象
```js
console.log(this instanceof Object);  // true
```
3. 作为全局变量的宿主
```js
var a = 10;
console.log(this.a)  // 10
```
> 此处需注意，只能使用`var` 声明变量，而不能使用`let`、`const`，因为只有`var` 声明的变量属于全局对象`Window`
### 函数上下文
当执行一个函数的时候，就会创建一个执行上下文，并且压入上下文栈。  
  
在函数作用域中，我们用活动对象（AO）来表示变量对象。  
  
活动对象在进入函数上下文的时候被创建，他通过函数的`arguments` 属性初始化。`arguments` 的属性值是`Arguments` 对象。
## 执行过程
执行上下文会分为两个阶段进行处理：**分析和执行**
1. 进入执行上下文
2. 代码执行
### 进入执行上下文
变量对象包括：
  1. 函数的所有形参（如果是函数上下文）
    - 由名称和对应值组成的变量对象的属性被创建
    - 如果没有实参，属性值设为`undefined`
  2. 函数声明
    - 由名称和对应值（函数对象(`function-object`)）组成一个变量对象的属性被创建
    - 如果变量对象已经存在相同的属性，则完全替代这个属性
  3. 变量声明
    - 由名称和对应值（`undefiend`）组成一个变量对象的属性被创建
    - 如果变量声明和已经声明的形式参数或函数相同，则变量声明不会干扰已经存在的这类属性
```js
function foo(a) {
  var b = 2;
  function c() {}
  var d = function() {};
  b = 3;
}

foo(1);
```
在进入执行上下文的时候，这时候创建的AO是：
```js
AO = {
  arguments: {
    0: 1,
    length: 1
  },
  a: 1,
  b: undefined,
  c: reference to function c(){},
  d: undefined
}
```
### 代码执行
在代码执行阶段，会顺序执行代码，根据代码，修改变量对象的值。  
  
当代码执行完时，这时候的AO应该是：
```js
AO = {
  arguments: {
    0: 1,
    length: 1
  },
  a: 1,
  b: 3,
  c: reference to function c(){},
  d: reference to FunctionExpression "d"
}
```
## 总结
1. 全局上下文的变量对象初始化是全局对象
2. 函数上下文的变量对象初始化只包括`Arguments` 对象
3. 在进入执行上下文时会给变量对象添加形参、函数声明、变量声明等初始的属性值
4. 在代码执行阶段，会再次丢该变量对象的属性值

## 思考题
```js
console.log(foo);

function foo(){
  console.log("foo");
}

var foo = 1;
```
会打印函数，而不是`undefined`  
  
这是因为在进行执行上下文的时候，**首先会处理函数声明，其次再处理变量声明，如果变量声明和已经声明的形式参数或函数相同，则变量声明不会干扰已经存在的这类属性**。
