# 基本数据类型
**基本数据类型的值是不可变的，比较是值的比较，引用数据类型的值是可变的，比较的是引用地址的比较。**  
  
**基本数据**类型存储在**栈**中，会**自动分配相对固定大小的内存空间，并由系统自动释放**。  
  
**引用数据**类型存储在**堆**中，**动态分配内存，内存大小不一，不是自动释放**。  
## null
**`Null` 特值没有对象，即该处不应该存在值**
### undefined与null的区别
```js
undefined === null;  // false
undefined == null;   // true

null === null;       // true
null == null;        // true

Number(null);        // 0
Number(undefined);   // NAN

isNaN(1 + null)      // false
isNaN(1 + undefined) // true
```
`null` 会被自动转换为`0`，所以又设计了`undefiend`  
  
> 注意：`==` 与`===` 前者会进行类型转换  
> `null` 是一个表示**无的对象**，转为数值时为`0`；`undefined` 是一个表示**无的原始值**，转为数值时为`NaN`。  
## undefined
**表示缺少值，就是此处应该有一个值，但是还没有定义**  
  
`undefined` 是全局对象的一个属性，也就是说，它是全局作用域的一个变量。  
  
`undefiend` 属性的属性特征：
| undefined属性的属性特征 |       |
| ----------------------- | ----- |
| writable                | false |
| enumerable              | false |
| configurable            | false |
  
> `undefiend` 可能在非全局作用域中被当作标识符来使用，因为`undefiend` 不是一个**保留字**
  
要**检查一个变量是否在全局上下文中存在可以通过检查全局对象上是否存在这个属性**
```js
if ('x' in window) {
  // 只有x被全局性的定义 才会这行这些语句
}
```
## BigInt
`BigInt` 是通过在整数末尾附加`n` 或**调用构造函数**来创建的。  
  
`BigInt` 不能与数字互换操作。否则，将抛出`TypeError`。
## Symbol
`Symbol` 是一种基本数据类型，`Symbol` 作为构造函数来说并不标准，因为他**不支持语法：`new Symbol()`**，这是因为生成的值是一个原始数据、而不是一个对象  
  
每个从`Symbol()` 返回的`symbol` 都是唯一的，一个`symbol` 能作为对象属性的标识符。  
  
基本语法：`Symbol([description])`
- `description` 可选的，字符串类型

```js
let symbol1 = Symbol('foo');
```
`Symbol('foo')` 不会强制字符串`'foo'` 转换为`symbol` 类型，他每次都会创建一个新的`symbol` 类型
