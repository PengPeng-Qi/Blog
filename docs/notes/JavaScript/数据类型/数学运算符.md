# 基础运算符
## 一元运算符 +
一元运算符加号，或者说，加号`+` 应用于单个值，对数字没有任何作用。但是如果运算元不是数字，加号`+` 则会**将其转化为数字**。  
  
```js
// 对数字无效
let x = 1;
alert( +x ); // 1

let y = -2;
alert( +y ); // -2

// 转化非数字
alert( +true ); // 1
alert( +"" );   // 0
```
> 它的效果和`Number(...)` 相同，但是更加简短。

```js
let apples = "2";
let oranges = "3";

// 在二元运算符加号起作用之前，所有的值都被转化为了数字
alert( +apples + +oranges ); // 5
```
> 一元加号运算符优先级高于二元加号运算符

## 原地修改
`+= -=`**这类运算符的优先级与普通赋值运算符的优先级相同，所以它们在大多数其他运算之后执行**
```js
let n = 2;
n *= 3 + 5;

alert(n); // 16 （右边部分先被计算，等同于 n *= 8）
```
`++/--` 运算符同样可以在表达式内部使用。**它们的优先级比绝大部分的算数运算符要高**。
```js
let counter = 1;
alert(2 * ++counter ); // 4
```
```js
let counter = 1;
alert(2 * ++counter ); // 4
```
## 逗号运算符
逗号运算符能让我们处理多个语句，使用`,` 将它们分开。每个语句都运行了，但是只有最后的语句的结果会被返回。
```js
let a = (1 + 2, 3 + 4);

alert(a); // 7（3 + 4 的结果）
```
> 逗号运算符的优先级非常低，比`=` 还要低

## && ||
与运算`&&` 在或运算`||` 之前进行  
与运算`&&` 的优先级比或运算`||` 要高。  
  
所以代码`a && b || c && d` 跟`&&` 表达式加了括号完全一样：`(a && b) || (c && d)`  
  
**非运算符`!` 的优先级在所有逻辑运算符里面最高，所以它总是在`&&` 和`||` 之前执行。**    
  
```js
let x = 1 || 2;
// x -> 1

let y = 1 < 2 || 2;
// y -> true
```
## ??
`??` 空值合并运算符、**如果第一个参数不是`null/undefined`，则`??` 返回第一个参数。否则，返回第二个参数**。
```js
result = a ?? b;

/* 等同于 */
result = (a !== null && a !== undefined) ? a : b;
```
> `||` 的优先级高于`??`，因为`??` 是最近才被添加到js中的  
> `??` 的优先级为5，`||` 为6，在`= ?` 运算前计算，但是在大多数其他运算(`+` `-`)之后计算

```js
let height = null;
let width = null;

// 重要：使用括号
let area = (height ?? 100) * (width ?? 50);

alert(area); // 5000
```
出于安全原因，JavaScript 禁止将`??` 运算符与`&&` 和`||` 运算符一起使用，除非使用括号明确指定了优先级。
```js
let x = 1 && 2 ?? 3; // Syntax error

let x = (1 && 2) ?? 3; // 正常工作了

alert(x); // 2

```
## 注意点
**禁止`break/continue` 在`?` 的右边**  
  
非表达式的语法结构不能与三元运算符`?` 一起使用。特别是`break/continue` 这样的指令是不允许这样使用的。  
```js
if (i > 5) {
  alert(i);
} else {
  continue;
}
```
```js
(i > 5) ? alert(i) : continue; // continue 不允许在这个位置、会显示有语法错误
```
