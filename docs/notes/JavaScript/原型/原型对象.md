# F.prototype
如果`F.prototype` 是一个对象，那么`new` 操作符会使用它为新对象设置`[[Prototype]]`  
  
```js
let animal = {
  eats: true
};

function Rabbit(name) {
  this.name = name;
}

Rabbit.prototype = animal;  // 当创建一个new Rabbit 时，把它的[[Prototype]] 赋值为 animal

let rabbit = new Rabbit("White Rabbit"); //  rabbit.__proto__ == animal

alert( rabbit.eats ); // true
```
## <code>__proto__</code>
`__proto__` 被认为是过时且不推荐使用的，`__proto__` 只在浏览器环境下才得到支持，现代的方法有：
- `Object.create(proto, [descriptions])` 利用给定的`proto` 作为`[[Prototype]]` 和可选的属性描述来创建一个空对象。
- `Object.getPrototypeOf(obj)` 返回对象`obj` 的`[[Prototype]]`
- `Object.setPrototypeOf(obj, proto)` 将对象`obj` 的`[[Prototype]]` 设置为`proto`
  
应该使用这些方法来代替`__proto__`。  
```js
let animal = {
  eats: true
};

// 创建一个以 animal 为原型的新对象
let rabbit = Object.create(animal);

alert(rabbit.eats); // true

alert(Object.getPrototypeOf(rabbit) === animal); // true

Object.setPrototypeOf(rabbit, {}); // 将 rabbit 的原型修改为 {}
```
`Object.create` 有一个可选的第二参数：**属性描述器**。我们可以在此处**为新对象提供额外的属性**。属性描述器的格式与**[属性标志和属性标志符](https://zh.javascript.info/property-descriptors)**一样。**注意该属性是不可枚举、修改、配置的**。  
```js {6,8}
let animal = {
  eats: true
};

let rabbit = Object.create(animal, {
  jumps: {
    value: true
  }
});

/*
  rabbit:
  - {jumps: true}
    - jumps: true
    - [[Prototype]]: Object
      - eats: true
*/
```
比`for...in`更强大的复制：
```js
let clone = Object.create(Object.getPrototypeOf(obj), Object.getOwnPropertyDescriptors(obj))
```
但是依旧是浅拷贝
## constructor
属性`constructor` 指向函数本身
```js
function Rabbit() {}

/* default prototype
   Rabbit.prototype = { constructor: Rabbit };
 */
```
当我们有一个对象，但不知道他使用了那个构造器（例如来自第三方库），并且我们需要创建一个类似的对象时，用这种方法就很方便。
```js {8}
function Rabbit(name) {
  this.name = name;
  alert(name);
}

let rabbit = new Rabbit("White Rabbit");

let rabbit2 = new rabbit.constructor("Black Rabbit");
```
如果我们将整个默认`prototype` 替换掉，那么其中就不会有`constructor`
```js
function Rabbit() {};

Rabbit.prototype = {
  jumps: true
};

let rabbit = new Rabbit();
alert(rabbit.constructor === Rabbit); // false
```
为了确保正确的`constructor`，我们可以选择添加/删除属性到默认`prototype`，而不是将整个覆盖
```js
function Rabbit() {}

Rabbit.prototype.jumps = true;
```
或者，手动重新创建`constructor` 属性
```js
Rabbit.prototype = {
  jumps: true,
  constructor: Rabbit
}
```
> 在常规对象上，`prototype` 没有什么特殊的

## 原型对象
一些方法在原型上可能会发送重叠，例如：`Array.prototype` 有自己的`toString` 方法来列举出来数组的所有元素并用逗号分隔每一个元素
```js
let arr = [1, 2, 3]
alert(arr); // 1,2,3 <-- Array.prototype.toString 的结果
```
`Object.prototype` 也有`toString` 方法，但是从`Array.prototype` 在原型链上更近，所以数组对象原型上的方法会被使用。
## 基本数据类型
字符串、数字、布尔值不是对象，当我们访问他们的属性的时候，那么临时包装器对象将会通过内建的构造器`String`、`Number`、`Boolean` 被创建。他们提供给我们操作字符串、数字、布尔值的方法后然后消失。  
  
这些对象是无线创建出来的，大多数引擎会对其进行优化。  
> `null`、`undefined` 没有对象包装器

## 从原型中借用
如果我们要创建类数组对象，则可能需要向其中复制一些`Array` 方法
```js
let obj = {
  0: "Hello",
  1: "world!",
  length: 2,
};

obj.join = Array.prototype.join;

alert( obj.join(',') ); // Hello,world!
```
`join` 的内部算法只关心正确的索引和`length` 属性，不会检查这个对象是否是真正的数组。许多内建方法都是这样。  
  
另一种方法就是通过将`obj.__proto__` 设置为`Array.prototype`，这样`Array` 中的所有方法都自动地可以在`obj` 中使用了。
## 属性描述符
通过`Object.defineProperty(obj, protoName, descriptor)` 添加的对象默认`writeable`等属性都是`false`  
  
如果obj已经有了某个属性，则默认`writeable`等值均为`true`  
### Object.defineProperties
```js
Object.defineProperties(obj, {
  prop1: descriptor1,
  prop2: descriptor2
  // ...
});
```
### Object.getOwnPropertyDescriptors
要一次获取所有属性描述符，我们可以使用`Object.getOwnPropertyDescriptors(obj)`  
  
`for...in` 会忽略`Symbol` 类型的属性，但是`Object.getOwnPropertyDescriptors` 返回包含`Symbol` 类型的属性在内的所有**属性描述符**
## 测试
### 测试一
```js {9}
function Rabbit() {}

Rabbit.prototype = {
  eats: true
};

let rabbit = new Rabbit();

Rabbit.prototype = {};

alert( rabbit.eats ); // true 
```
`Rabbit.prototype` 的赋值操作作为新对象设置了`[[prototype]]`， 但他不影响已有的对象
### 测试二
```js {8}
function Rabbit() {}
Rabbit.prototype = {
  eats: true
};

let rabbit = new Rabbit();

Rabbit.prototype.eats = false;

alert( rabbit.eats ); // false
```
对象通过引用被赋值，来自`Rabbit.prototype` 的对象并没有被赋值，他仍然是被`Rabbit.prototype` 和`rabbit` 的`[[Prototype]]` 引用的单个对象。
