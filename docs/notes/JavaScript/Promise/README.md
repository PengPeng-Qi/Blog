# Promise
## 优点
> 一般异步操作：先指定回调函数、再启动异步任务  

**Promise优点**：

1. 指定回调函数的方式更加灵活，**可以先启动异步任务，后指定回调函数**。  
2. **支持链式调用，解决地狱回调(回调函数嵌套调用)问题**。  

**回调地狱的问题**：

1. 不便于阅读
2. 处理问题麻烦

`.catch()` 异常传透

> 1、Promise 并不是终极选择，因为还存在回调函数  
> 2、`async/await` 无回调函数

## Promise.all
`Promise.all(parmas)` 传递一个Promise实例对象的数组作为参数，返回一个Promise对象。  
  
如果有一个失败，则返回的Promise对象为失败，且值是失败的Promise实例传递的值，如果一个Promise失败，则直接返回，不需要再看之后的promise的状态，故如果多个Promise的状态为失败，reason的值也是第一个失败状态的Promise传递的值。  

如果全部成功，则value的值为成功的Promise实例的值组合成的数组。且顺序为参数数组的顺序，不会因为状态改变先后而改变顺序。  
## 注意点
### 注意点一
如果一个promise 在executor函数中抛出一个异常`throw new Error('出错了')` ，则返回的Promise 对象的状态变为`rejected`，reason为抛出的错误对象。  
### 注意点二
如果一个Promise对象指定多个成功或失败的回调函数，则Promise改变为对于的状态时都会调用。
### 注意点三
**改变Promise状态和指定回调函数先后顺序？**  
  
1、都有可能，正常情况下是先指定回调函数，再改变状态，但是可以先改变状态再执行回调函数。  
  
2、先改变状态，再执行回调函数。在执行器中调用`resolve()` 或`reject()`，然后延长时间再调用`.then()`  
  
**什么时候才能得到数据？**  
  
1、如果先指定回调函数，则状态一改变，回调函数就会调用，然后获得数据。  
2、如果先改变状态，那指定回调函数时，回调函数就会调用，然后获得数据。
