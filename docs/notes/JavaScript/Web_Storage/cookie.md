# cookie
`cookie` 用于保存状态以及能够为Web 浏览器提供一种**身份识别机制**。  
  
cookie**不会采用任何的加密机制**，因此他们是不安全的。  
  
> 通过`https` 传输`cookie` 是安全的，不过与`cookie` 本身无关

任何以`cookie` 形式存储的数据，不论服务器端是否需要，每一次HTTP 请求都会把这些数据传输到服务器端。  
## 有效期
一旦用户关闭浏览器，cookie保存的数据就丢失了。  
  
`cookie` 的作用域**并不局限在浏览器的单个窗口中**，他的有效期和整个浏览器进程而不是单个浏览器窗口的有效期一致。  
  
如果想要延长`cookie` 的有效期，可以通过设置`max-age` 属性（单位为秒）  
## 作用域
`cookie` 的作用域通过文档源和文档路径来确定。可使用`path` 和`domain` 来进行配置。  
    
默认情况下，`cookie` 和创建它的Web页面有关，对该页面和该Web页面的同目录或者子目录的其他Web页面创建一个`cookie`。  
## 局限性
浏览器保存的`cookie` 不能超过300 个，单个`cookie` 的数据不能超过4KB
