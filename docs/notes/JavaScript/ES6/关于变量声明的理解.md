## Const连续赋值
```js
const A = 1, B = 2, C = 3;

A = 4;  // Uncaught TypeError: Assignment to constant variable.
B = 5;  // Uncaught TypeError: Assignment to constant variable.
C = 6;  // Uncaught TypeError: Assignment to constant variable.
```
> 同理：`Let`、`Var` 的连续声明都如此

## 变量声明
如果一个变量被声明赋值，当值改变的时候，之前的数据就被从变量中删除了
```js
let message;

message = 'Hello!';
message = 'World!'; // 值改变了
```
## 严格模式
```js
num = 5;          // 如果变量 "num" 不存在，就会被创建

console.log(num); // 5
```
```js
"use strict";
num = 5;        

console.log(num);  // Uncaught ReferenceError: num is not defined
```
## 函数作用域
```js {4}
let userName = 'John';

function showMessage() {
  userName = "Bob"; // 改变外部变量

  let message = 'Hello, ' + userName;
  alert(message);
}

alert( userName ); // John 在函数调用之前
showMessage();

alert( userName ); // Bob，值被函数修改了
```
```js {4}
let userName = 'John';

function showMessage() {
  let userName = "Bob"; // 声明一个局部变量

  let message = 'Hello, ' + userName; // Bob
  alert(message);
}

// 函数会创建并使用它自己的 userName
showMessage();

alert( userName ); // John，未被更改，函数没有访问外部变量。
```
