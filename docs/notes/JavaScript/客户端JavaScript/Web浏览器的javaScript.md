# JavaScript
## 同步、异步和延迟的脚本
脚本的执行只在默认情况下是同步和阻塞的。  
  
`<script>` 标签可以有`defer` 和 `async` 属性。  
  
这些都是布尔属性，没有值。  
  
`defer` 属性使得浏览器**延迟脚本的执行**，这个布尔属性被设定用来通知浏览器该脚本将在文档完成解析后，触发`DOMContentLoaded` 事件前执行。。  
  
`async` 属性使得浏览器可以**尽快执行脚本**，而不用在下载脚本时阻塞文档解析。遇到`async` 属性时，开始下载脚本文本，继续解析文档。脚本会在下载完成后尽快执行，解析器没有停下来等他下载。异步脚本禁止使用`document.write()` 方法。  
  
如果两个属性同时存在，浏览器会遵循`async` 属性并忽略`defer` 属性。  
## JavaScript不能做什么
1、没有权限写入或删除客户计算机上的任意文件或列出任意目录  
2、在已有的功能上加一些限制
