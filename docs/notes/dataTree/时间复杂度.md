# 时间复杂度
时间复杂度分为**多项式量级**和**非多项式量级**  
  
**非多项式量级**只有两个：<code>O(2<sup>n</sup>) 和 O(n!)</code>
  
**当数据规模 n 越来越大时，非多项式量级算法的执行时间会急剧增加**，求解问题的执行时间会无限增长。所以，非多项式时间复杂度的算法其实是非常低效的算法。因此，关于 非多项式量级时间复杂度我就不展开讲了。我们主要来看几种常见的**多项式时间复杂度**。
## 多项式量级
- 常量阶：`O(1)` 只要代码的执行时间不随n增大，则为常量阶
- 对数阶：`O(logn)` 常见在`while` 循环中
```js
while (i <= n) { 
  i = i * 2; 
}
```
因为<code>2<sup>1</sup>  2<sup>2</sup>  ...  2<sup>x</sup> = n</code>，<code>x = log2<sup>n</sup></code>
- 线性阶：`O(n)`
- 线性对数阶：`O(nlogn)`
- 平方阶：<code>O(n<sup>2</sup>)</code>
  
**时间复杂度**：表示**算法的执行时间与数据规模之间**的增长关系  
**空间复杂度**：表示**算法的存储空间与数据规模之间**的增长关系  
  
越高阶复杂度的算法，执行效率越低，从低阶到高阶有：`O(1)`、`O(logn)`、`O(n)`、`O(nlogn)`、<code>O(n<sup>2</sup>)</code>
