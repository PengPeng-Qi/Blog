# 链表
数组需要**一块连续的内存空间**来存储。  
  
链表**不需要一块连续的内存空间**，他通过**指针将一组零散的内存块串联**起来使用。
  
链表分为：**单链表、双向链表、循环链表**
  
## 单链表
我们把内存块称为链表的**结点**，为了将所有的节点串起来，每个链表的结点除了存储数据之外，还需要记录链上的下一个结点的地址。继续下一个结点地址的指针叫做**后继指针`next`**  
  
第一个结点叫作**头结点**、最后一个结点叫作**尾结点**。  
  
头结点用来记录链表的基地址。有了它，我们可以遍历整条链表。而尾结点特殊的地方是：指针不是指向下一个结点，而是指向一个空地址`NULL`，表示这是链表上最后一个结点。  
  
访问结点需要一个一个的遍历，直到找到相应的结点。  
  
> 链表的插入、删除，我们只需要考虑相邻结点的指针变化，所以对应的时间复杂度是`O(1)`。  
## 循环链表
**单链表**的尾结点指向空地址，**循环链表**的尾结点指向链表的头结点

> 当要处理的数据具有环形结构特点时，就特别适合采用循环链表。

## 双向链表
**单向链表**只有一个方向，结点只有一个后继指针`next` 指向后面的结点。  
  
**双向链表**支持两个方向，每个结点不止有一个后继指针`next` 指向后面的结点，还有一个前驱指针`prev` 指向前面的结点。    
  
双向链表需要两个额外的空间来存储后继结点和前驱结点的地址。    
  
双向链表支持`O(1)` 的时间复杂度的情况下找到前驱结点。  
  
> 使用较多

## 设计思想
### 空间换时间
内存空间充足的时候，如果更加追求代码的执行速度，我们就可以选择**空间复杂度相对较高、时间复杂度相对较低的算法**。  

> 缓存：把数据存储在硬盘上，节省内存，但是每次查找数据都需要遍历硬盘，故会比较慢，所以需要采取空间换时间的思想
### 时间换空间
> 如果针对消耗过多的内存的程序，可以通过时间换空间来降低内存的消耗。
## 数组&链表
1、链表没有大小的限制，支持动态扩容  
2、对链表的频繁插入、删除操作，会导致频繁的内存申请和释放，链表的每个节点都需要消耗额外的存储空间去存储一份指向下一个节点的指针
| 时间复杂度 | 数组 | 链表 |
| ---------- | ---- | ---- |
| 插入删除   | O(n) | O(1) |
| 随机访问   | O(1) | O(n) |

数组，连续的内存空间，可以借组CPU的缓存机制，预读数组中的数据，所以访问效率更高。  
  
**数组的缺点**：大小固定，一经过声明就要占用整块连续的内存空间。
### LRU缓存淘汰算法
首先维护一个有序链表，越靠近链表尾部的结点说明是越早之前访问的。(越靠近前面，表示越新)  
  
当有一个新的数据访问时，需要从链表头开始顺序遍历链表，此时会有以下这几种情况：  
- 如果此数据之前**已经被缓存在链表**中了，我们遍历得到这个数据对应的结点，并将其从原来的位置删除，然后再插入到链表的头部。
- 如果此数据**没有在缓存链表**中，又可以分为两种情况：
  - 如果此时缓存未满，则将此结点直接插入到链表的头部；
  - 如果此时缓存已满，则链表的尾结点删除，将新的数据结点插入链表的头部。

## 技巧
### 指针与引用
`p->next=q` 指的是p结点中的`next` 指针存储了q结点的内存地址。  
`p->next=p->next->next` p结点的`next` 指针存储了p结点的下下一个结点的内存地址。  
### 警惕指针丢失和内存泄漏
删除链表结点时，记得手动释放内存空间。如果没有手动释放对应的内存空间，则会产生**内存泄漏**。  
### 利用哨兵简化实现难度
单链表插入结点：
```js
new_node->next = p->next;
p->next = new_node;
```
如果要向一个空链表插入第一个结点，刚刚的逻辑就不适用了，`head` 表示链表的头节点。
```js
if (head === null){
  head = new_node;
}
```
删除结点：
```js
p->next = p->next->next;
```
删除最后一个结点：
```js
if (head->next === null){
  head = null;
}
```
> 针对链表的删除、插入操作，需要对插入第一个结点和删除最后一个结点的情况进行特殊处理。

**哨兵**，解决的就是国家的边界问题。  
  
引入哨兵结点，不管链表为不为空，head指针都会一直指向这个哨兵结点。我们也把这种有哨兵结点的链表叫做**带头链表**。  
  
> 哨兵结点不存储数据，因为哨兵结点一直存在。

### 留意边界条件处理
- 1、如果链表为空，代码是否正常运行
- 2、如果链表只包含一个结点，代码是否正常运行
- 3、如果链表只包含两个结点，代码是否正常运行
- 4、如果链表在处理头结点和尾结点的时候，代码是否正常运行
