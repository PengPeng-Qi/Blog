# HTTP
## OSI模型  
物理层 -> 数据链路层 -> 网络层 -> 传输层 -> 会话层 -> 表示层 -> 应用层  
## TCP/IP协议族  
链接层 -> 网络层 -> 传输层 -> 应用层
## HTTP
HTTP是一个**无状态的应用层协议**、基于TCP协议

- HTTP/0.9 只支持GET请求
- HTTP/1.0 增加了Header、有了状态码、支持各种文档类型
- HTTP/1.1 链接复用、缓存、内容协商
- HTTP/2   二进制协议、压缩header、服务器推送

### Methods
| 方法    | 描述                                              |
| ------- | ------------------------------------------------- |
| GET     | 请求一个指定资源的表现形式，只能用于获取数据      |
| HEAD    | 请求一个与GET请求的响应相同的响应，不过没有响应体 |
| POST    | 将实体提交给指定的资源，数据包含在请求体中        |
| PUT     | 用请求有效载荷替换当前目标资源的所有当前表示      |
| DELETE  | 删除指定的资源                                    |
| CONNECT | 建立一个到由目标资源标识的服务器的隧道            |
| OPTIONS | 用于描述目标资源的通信选项                        |
| TRACE   | 回显服务器收到的请求，主要用于测试或诊断          |
| PATCH   | 是对PUT方法的补充，用来对已知资源进行局部更新     |

Methods分类：
- 安全的：不会修改服务器的数据的方法：GET、HEAD、OPTIONS
- 幂等：一次请求和重复的多次请求对系统资源的影响是一致的，GET、HEAD、OPTIONS、PUT、DELETE

### 状态码
| 状态码              | 描述                                                 |
| ------------------- | ---------------------------------------------------- |
| 200 OK              | 客户端请求成功                                       |
| 301                 | 资源被永久转移到其他URL                              |
| 302                 | 临时跳转                                             |
| 401 Unauthorized    | 请求未经授权                                         |
| 404                 | 请求的资源不存在                                     |
| 500                 | 服务器内部发生了不可预期的错误                       |
| 504 Gataway Timeout | 网关或者代理的服务器无法在规定的时间内获得想要的回应 |

### 缓存
**强缓存**：如果缓存在有效期内，必须使用缓存  
**协商缓存**：客户端和服务端每次都沟通一下是否可以使用
