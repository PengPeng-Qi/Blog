# 错误边界
**错误边界**(Error boundary)：用来捕获后代组件错误，渲染出备用页面
## 特点
只能**捕获后代组件生命周期产生的错误**，不能捕获自己组件产生的错误和其他组件在合成事件、定时器中产生的错误
## 使用方式
`getDerivedStateFromError` 配合`componentDidCatch`
```js
/* 
  生命周期函数，一旦子组件报错，就会触发，并且携带错误❌信息
  getDerivedStateFromError()生产环境下使用会一闪而过
  npm build之后就会很稳定
 */

state = {
  hasError: ''
}

static getDerivedStateFromError(error) {
    console.log(error);
    return {
        hasError: true,
    };
}
/* hasError的值可以用来判断是否渲染错误组件或者其他替代组件 */

/* 如果子组件出现问题就会调用componentDidCatch() */
componentDidCatch(error, info) {
    /* 统计页面的错误，反馈给服务器 */
    console.log(error, info);
}
```
